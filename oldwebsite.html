F L E X B O X

ABOUT FLEXBOX

Flexbox is a layout module thatâ€™s goal is to provide an efficient way to arrange elements on a page. These elements are in a container and it is possible to layout, align, and distribute space among the items in the container without knowing their size. The main purpose of the layout is allowing the container to best fill the space provided, whether it be on a computer page or mobile screen. A flex container can expand or shrink items to either fill available space or prevent overflow. Thus, Flexbox is responsive and its current uses reflect its responsive capabilities.

HISTORY

Overview

The first working flexbox draft came out July 23, 2009. There were three major revisions of the flexbox spec. The current version is stable and a W3C (World Wide Web Consortium) Candidate Recommendation. The three major revisions of the flexbox spec are: the 2009 working draft, the 2011 version, and the current working draft version from May 2015. The current working draft was reviewed through June 11, 2015 and there is now an editor's draft as of October 2015.

Issue #12
Summary: Should consider 'stack' as a flexbox direction.

Proposal: flex-flow: stack;

Resolved: This suggestion is not appropriate for flexbox, due to the alignment being only applicable to one direction at a time. Grid layout behaves as stack when rows/columns are not specified for items.

Issue #13:
Summary: The default values in flex() function should be the initial values. The current spec says "If flex() function has a single value and it is a length, a percentage, or a valid keyword for 'width' or 'height', the preferred size is set to that value, the positive flexibility is set to 1, and the negative flexaibility is set to 0."

Proposal: Should always use the initial values as defaults in flex().

Resolved: The default values in flex() function will be different from initial because it is more useful to have default flex of 1 rather than 0. Also, using flex() function makes it clear that flexibility is desired.

Issue #14:
Summary: What is the meaning of 'before' and 'after' in 'flex-align'? The 2009 spec reversed alignment direction for reverse flexbox but that did not make sense. There is no way to specify alignment direction without having to specifiy wrap.

Proposal: Make alignment direction wrap direction in multi-line flexbox and block flow direction in single-line flexbox.

Flex margins should either have full support of flexbility or none at all. Suggesting if 'auto' margins on main axis were used after flex sizing and before Pack.

Proposal: Apply to 'width' and 'height' only.

Resolved: Resolved as proposed.

Issue #16:
Summary: There needs to be a solid use case for treating "margin:auto" as "margin:flex(0px 1 0)" along main axis in flexbox.

Proposal: Remove flexible margins.

Resolved: Resolved as proposed.

Issue #17:
Summary: What is the effective min width/height of a flex item with specified width/height? Normally the min-width is irrelevant when width is specified but flex item can be smaller than specified size due to negative flexing.

Option 1: Use content min-width when sizing to content, specified min-width otherwise; min-height only if specified.

Option 2: Use min-width only if specified.

Option 3: min-width/min-height is consulted after flex distribution. If any item is smalelr than min, it gets its min size and no flexibility, then flex distribution is restarted.

Proposal: Use Option 3: use of min-width vs min-content to be clarified.

Resolved: Minimum width is just min-width: min-content is not an implied minimum (min-content is never taken into account in flex calculations unless used explicity).

Issue #18:
Summary: The combination of margins and item-level flex-align is redudant and is not support common case where all items have the same alignment.

Proposal: Have auto margins set to zero or auto margins behave as horizontal margins in normal flow (align or center).

Resolved: Accepted the proposal.

TERMINOLOGY

Definitions

display- defines a flex container; inline or block depending on the given value. It allows a flex context for all of its direct children.

See the Pen flex display by Anna Irish (@AnnaIrish) on CodePen.

flex direction- Defines the direction flex items are placed in the flex container. Flexbox is a single-direction layout concept. Flex items have primarily horizontal rows or vertical columns.

See the Pen flex direction by Anna Irish (@AnnaIrish) on CodePen.

flex-wrap- by deafault, flex items are single-lined. 'flex-wrap' allows the items to wrap as needed. The direction in which the new lines are stacked can also be determined.

See the Pen flex-wrap by Anna Irish (@AnnaIrish) on CodePen.

flex-flow- This is the shorthand for flex-direction and flex-wrap properties, which when combined define the flex container's main and cross axes. Default is row nowrap.

See the Pen flex-flow by Anna Irish (@AnnaIrish) on CodePen.

justify-content- Defines the alignment along the main axis. Helps distribute extra free space left over when either all the flex items on a line are inflexible or are flexbile but have reached their maximum size.

See the Pen justify-content by Anna Irish (@AnnaIrish) on CodePen.

align-items- Defines the default behaviour for how flex items are laid out along the cross axis on the current line.

See the Pen align-items by Anna Irish (@AnnaIrish) on CodePen.

align-content- Aligns a flex container's lines within where there is extra space in the cross-axis. This property has no effect when there is only one line of flex items.

order- Controls the order in which flex items appear in the flex container.

flex-grow- Defines the ability for a flex item to grow if necessary. Accepts unitless value that serves as a proportion and dictates what amount of the available space inside the flex container the items should take up.

flex-shrink- Defines ability of a flex item to shrink if necessary.

flex-basis- Defines the default size of an element before the remaining space is distributed.

flex- Shorthand for 'flex-grow', 'flex-shrink', and 'flex-basis' combined. Shorthand is recommended because it sets the other values intelligently.

align-self- Allows the default alignment (or the one specified by 'align-items')to be overridden for individual flex items.

Grid vs. Flexbox

Grid

How the page renders: When using grid, the layout is defined in the container therefore the nav is rendered into the middle column as soon as it starts to arrive.

Arranging content: Grid can only arrange the content in the order it is coded.

Flexbox

How the page renders: When using the flexbox module, content dictates the layout of the webpage. This is because the container declares itself as a flexbox and the child elements declare how they will interact with each other inside the flexbox. When the page starts to load, the container begins receiving the first child, which is the main content. When the nav arrives, the content resizes to make room for it.

Arranging content: By default, Flexbox will arrange items in source order. However, the order property controls how they actually appear in the flex container.

Browser Support

The newest spec is implemented in all 5 major desktop browsers: Chrome, Firefox, Safari, Internet Explorer 10+, and Opera 12.1+. There are no robust polyfills for the current spec, but you can mix old and new flexbox for maximum browser support. https://css-tricks.com/using-flexbox/
